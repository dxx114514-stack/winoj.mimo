# WinOJ - Windows 在线评测系统

专为 Windows 平台设计的在线评测系统，支持多语言代码评测、安全沙箱执行、自定义计分脚本、比赛管理、文章系统、Rating 机制等功能。

## 功能特性

### 评测核心
- **多语言支持**：C、C++、Python 3、Java、JavaScript，可通过管理面板动态添加/禁用语言
- **安全沙箱**：代码在隔离临时目录中执行，CPU/内存/IO 受限，Windows 下通过原生 memwatch.exe 监控峰值内存
- **多种比较模式**：严格文本比较、宽松文本比较、浮点数容差比较、Special Judge
- **自定义计分脚本**：支持变量、算术运算、位运算、逻辑运算、条件分支，可针对测试点组或整题自定义评分
- **子任务系统**：测试点分组、组间依赖、自定义聚合策略（求和/取最小/取最大等）
- **编译错误检测**：编译失败时提前终止评测，返回详细编译输出

### 题目管理
- **题目 CRUD**：教师及以上角色可创建、编辑、删除题目
- **Markdown 支持**：题目描述、输入格式、输出格式、提示均支持 Markdown + KaTeX/LaTeX 公式
- **提供者标注**：可标注题目提供者，超级管理员可修改
- **测试点管理**：支持内联添加、批量添加、文件上传(.in/.out)、编辑、删除
- **测试分组**：创建 Subtask，设置满分、聚合方式、依赖关系、计分脚本
- **题目级计分脚本**：覆盖默认聚合策略，自定义整题评分逻辑

### 在线编程 (IDE)
- **浏览器代码编辑器**：基于 CodeMirror，支持语法高亮、行号
- **即时运行**：支持自定义 stdin 输入，显示 stdout/stderr/退出码/耗时
- **免登录使用**：访客也可使用 IDE

### 用户与权限
- **四级角色体系**：用户 → 教师 → 管理员 → 超级管理员
- **用户资料**：个性签名（支持 `D("内容")` 动态签名）、个人简介（Markdown）
- **Rating 等级分**：编译错误 -2 Rating，新 AC +10 Rating，超级管理员可任意设置
- **封禁系统**：封禁时同时踢出登录，仅阻止下次登录
- **强制登出**：删除 refresh token，最长 15 分钟后自动失效
- **免密登录**：超级管理员可一键切换到任意用户身份
- **在线用户列表**：管理员可查看 5 分钟内活跃的在线用户
- **Token 过期检测**：页面加载时自动检测 JWT 过期，清除失效登录态

### 比赛系统
- **比赛 CRUD**：教师及以上角色创建、编辑、删除比赛
- **题目关联**：将题目添加到比赛（含非公开题目），支持自定义别名和排序
- **邀请制参赛**：教师可邀请任意用户加入比赛，支持手动加入
- **比赛排行榜**：按总分降序、耗时升序排列，前三名金银铜牌标识
- **状态自动判断**：未开始 / 进行中 / 已结束
- **题解自动发布**：比赛结束后可自动发布关联题解

### 文章系统
- **文章 CRUD**：教师及以上角色可创建、编辑、删除文章
- **Markdown 支持**：文章内容支持完整 Markdown 语法
- **题解链接**：可将文章链接到题目作为题解
- **权限控制**：教师及以上或单独授予权限的用户可发布

### 文件与图床
- **文件上传**：支持图片、文档等文件上传（最大 10MB）
- **图床功能**：上传文件可通过 URL 访问，方便在 Markdown 中插入资源
- **文件管理**：记录上传者、文件名、大小等信息

### 提交管理
- **提交列表**：支持按用户、题目、状态筛选
- **提交详情**：显示详细错误说明（编译错误、运行错误、超时原因等）
- **重测功能**：教师及以上可一键重测任意提交
- **删除提交**：管理员可删除任意提交记录
- **Rating 自动计算**：评测完成自动更新用户 Rating

### 首页统计
- 实时显示题目数、提交数、用户数、语言数

### 安全机制
- **双 Token 认证**：15 分钟 Access Token + 7 天 Refresh Token（HttpOnly Cookie）
- **速率限制**：提交和 IDE 运行接口均有令牌桶限流
- **输入校验**：SQL 注入防护、源码大小限制、语言可用性检查
- **沙箱隔离**：用户代码在独立临时目录执行，结束后自动清理
- **封禁仅限登录**：封禁用户不影响已有会话，仅阻止下次登录

## 快速开始

### 方式一：一键启动（推荐）

双击项目根目录的 `start.bat` 脚本，自动完成依赖安装、服务器启动和浏览器打开。

### 方式二：手动启动

```bash
cd backend
npm install
npm start
```

启动后访问 http://localhost:3000

### 默认管理员账户

- **用户名**：`admin`
- **密码**：`admin123`

> **重要**：首次登录后请立即修改管理员密码！

## 配置说明

编辑 `backend/config/config.js`：

| 配置项 | 默认值 | 说明 |
|--------|--------|------|
| `port` | 3000 | 服务器端口 |
| `jwt.accessSecret` | 内置默认值 | Access Token 签名密钥 |
| `jwt.refreshSecret` | 内置默认值 | Refresh Token 签名密钥 |
| `jwt.accessExpiry` | `15m` | Access Token 过期时间 |
| `jwt.refreshExpiry` | `7d` | Refresh Token 过期时间 |
| `database.path` | `../data/winoj.db` | SQLite 数据库路径 |
| `sandbox.tempDir` | 系统临时目录 | 沙箱工作目录 |
| `sandbox.maxOutputSize` | 64KB | 最大输出大小 |
| `sandbox.maxSourceSize` | 64KB | 最大源码大小 |
| `rateLimit.submissions` | 10次/分钟 | 提交频率限制 |
| `rateLimit.ideRun` | 20次/分钟 | IDE 运行频率限制 |

## 项目结构

```
winoj/
├── start.bat               # 一键启动脚本
├── README.md               # 项目说明
├── APIuse.md               # API 接口文档
├── backend/
│   ├── config/             # 配置文件
│   ├── database/           # 数据库 schema 和初始化（含自动迁移）
│   ├── middleware/          # 认证、限流、在线用户追踪
│   ├── routes/             # API 路由（9个模块，60+端点）
│   ├── services/           # 评测引擎（含 Rating 自动计算）
│   ├── sandbox/            # 代码执行沙箱 + 计分脚本解释器 + memwatch
│   └── src/                # 服务器入口
├── frontend/
│   ├── css/                # 全局样式（左对齐等）
│   ├── js/                 # 公共 JS（API 封装、Markdown 渲染、导航栏）
│   └── pages/              # HTML 页面（全中文界面，支持 Markdown/KaTeX）
├── scripts/                # 工具脚本（种子数据等）
├── data/                   # 运行时数据（自动创建）
└── problems/               # 测试点文件（自动创建）
```

## 常见问题

**Q: 代码提交后一直显示"评测中"？**
A: 检查服务器控制台是否有报错。可能是编译器未安装或路径未配置。

**Q: 内存显示 0KB？**
A: 内存监控依赖 Windows 原生 memwatch 工具，仅在 Windows 下可用。确保 `sandbox/memwatch.exe` 存在。

**Q: 如何添加新语言？**
A: 以超级管理员登录，进入"语言"管理页面，填写编译/运行命令即可。

**Q: 自定义计分脚本怎么写？**
A: 使用 `@变量` 赋值、`if/fi` 条件判断、`==`/`and`/`or` 运算符。详见 API 文档中计分脚本部分。

**Q: 如何在题目描述中使用公式？**
A: 行内公式用 `$E=mc^2$`，块级公式用 `$$\sum_{i=1}^{n} i$$`。

**Q: 数据库在哪里？**
A: 默认在 `backend/data/winoj.db`，SQLite 格式，所有数据在重启后保留。
